# 発着駅選択画面仕様

## 対象
- ルート: `/terminal-selection`
- 実装: `src/routes/terminal-selection/+page.svelte`

## 画面の役割
- 発駅または着駅候補を選ぶ入口画面。
- グループ、都道府県、履歴、検索という複数の探索手段を 1 画面に集約する。

## 画面モード
- `mode=start`
  - 発駅選択モード
  - 選択結果は新しい `Farert` の開始駅になる
- `mode=destination`
  - 着駅選択モード
  - 選択結果は `autoRoute()` の終点入力になる

画面タイトルはモードで切り替える。

## 内部状態
- `tab`: `group` / `prefecture` / `history`
- `stage`: `root` / `lines` / `stations`
- `selectionBase`: `group` / `prefecture`
- `searchMode`: 検索 UI 表示中か
- `searchQuery`: 検索文字列
- `selectedCompany`, `selectedPrefecture`, `selectedLine`
- `historyItems`: 駅履歴

### 状態遷移の原則
- `searchMode=true` の間は `tab` / `stage` より検索結果表示を優先する。
- タブ切替時は検索状態、選択済み路線、駅一覧メタデータを破棄する。
- `selectedLine` は狭幅では `stage=stations` の文脈、広幅では右ペイン表示中の文脈を兼ねる。
- 駅確定時に route へ渡す駅名は、一覧の元データではなく解決済み表示名を使う。

## 画面構成

### ヘッダー
- 戻る
- 画面タイトル
- 必要に応じて履歴編集導線

### タブ列
- グループ
- 都道府県
- 履歴

### 検索バー
- あいまい検索入力
- クリア操作
- 検索中のローディング状態

### リスト領域
- `stage=root`
  - グループ一覧、都道府県一覧、履歴一覧のいずれか
- `stage=lines`
  - 選択基準に紐づく路線一覧
- `stage=stations`
  - 選択路線に属する駅一覧
- `searchMode=true`
  - タブ状態より優先して検索結果一覧を表示
- 画面横幅が十分に広い場合は、`stage=lines` で路線一覧と駅一覧の 2 ペイン表示に切り替えてよい
- このとき右ペインの駅一覧は、選択中路線に対する `stage=stations` 相当の内容を同一画面内に表示する

## データ取得
- グループ一覧: `getCompanys()`
- 都道府県一覧: `getPrefects()`
- グループ別路線: `getLinesByCompany(company)`
- 都道府県別路線: `getLinesByPrefect(prefecture)`
- グループ配下駅一覧: `getStationsByCompanyAndLine(company, line)`
- 都道府県配下駅一覧: `getStationsByPrefectureAndLine(prefecture, line)`
- 路線配下駅一覧: `getStationsByLine(line)`
- あいまい検索: `searchStationFuzzy(keyword, 50)`
- 補助情報:
  - `getKanaByStation(station)`
  - `getPrefectureByStation(station)`
  - `getLinesByStation(station)`
  - 同名駅補助情報は `executeSql()` の `samename` 参照で補完してよい

### 取得データの解釈
- `getPrefects()` の返却が不完全でも、既知の都道府県セットを優先して補完表示してよい。
- 路線一覧・駅一覧 API は JSON 配列だけでなく、最初の配列値を持つオブジェクトも許容してよい。
- JSON 解析失敗時は空一覧として扱い、画面内エラーメッセージで通知する。

### 都道府県起点の駅一覧解決
- まず `getStationsByPrefectureAndLine(prefecture, line)` を使う。
- 返却が不正または空でも、必要なら `getStationsByLine(line)` と各駅の `getPrefectureByStation(station)` を使って絞り込み直してよい。
- したがって別プラットフォーム実装では「都道府県別 API が常に完全」と仮定してはならない。

## 駅一覧表示
- 路線配下の駅一覧は 2 行表示を基本とする。
- 1 行目は表示駅名とし、同名駅がある場合はサフィックス付きで表示する。
  - 行の右端には都道府県を表示してよい。
  - ただし直前行と同じ都道府県が続く場合は、その行の都道府県表示を省略する。
  - 例: `追分(室)`
  - 例: `千歳(千)`
- 2 行目はかなを丸括弧付きで表示する。
  - 例: `（あやし）`
- 同一駅が複数路線に所属する場合だけ、2 行目のかなに続けて `/` 区切りで所属路線一覧を表示する。
  - 例: `（おいわけ）/室蘭線/石勝線`
- 所属路線が 1 件だけの駅は、2 行目に路線名を表示しない。
- かな取得と所属路線取得は、表示駅名に使った同名駅識別子込みの駅名で問い合わせる。

### 表示駅名の決定手順
1. 元の駅一覧データを受け取る。
2. 各駅についてベース名を `normalizeStationName()` 相当で求める。
   - 末尾の同名駅サフィックスを除去する
   - `追分(室)` -> `追分`
   - `追分((奥))` -> `追分`
3. `executeSql()` で `samename` を問い合わせる。
   - まず現在路線で絞り込む
   - 見つからなければ路線条件なしで再照会する
4. 取得した `samename` は生データ起源の括弧付き文字列として扱う。
   - 典型値は `(室)` のような形式
5. 表示名解決ユーティリティが括弧重複を吸収し、単一の丸括弧へ正規化する。
   - `(室)` / `((室))` はどちらも `(室)` として扱う
6. 元データに既にサフィックス付き駅名が入っている場合も、そのまま表示せず正規化する。
   - `追分((奥))` -> `追分(奥)`
7. 確定用駅名、可視ラベル、アクセシブル名には同じ解決済み表示名を使う。

### UI 実装原則
- 画面コンポーネントは `samename` に対して `(${suffix})` のような文字列結合を直接書かない。
- 括弧の有無や重複括弧の吸収は共通の表示名解決ユーティリティの責務とする。
- 別プラットフォーム実装でも同じ責務分離を維持すること。

### 駅一覧メタデータの取得順
- `kana`:
  1. 解決済み表示名で `getKanaByStation(displayName)`
  2. 失敗時のみベース名で `getKanaByStation(baseName)`
- `lines`:
  1. 解決済み表示名で `getLinesByStation(displayName)`
  2. 空または失敗時のみベース名で `getLinesByStation(baseName)`
- 取得した `lines` は重複除去したうえで表示する。

## 操作フロー

### グループ起点
1. JR グループを選ぶ
2. 路線一覧を表示する
3. 路線を選ぶ
4. 狭幅時は駅一覧画面へ進み、広幅時は同一画面右ペインに駅一覧を表示する
5. 駅を確定する

### 都道府県起点
1. 都道府県を選ぶ
2. 路線一覧を表示する
3. 路線を選ぶ
4. 狭幅時は駅一覧画面へ進み、広幅時は同一画面右ペインに駅一覧を表示する
5. 駅を確定する

### 履歴起点
1. 履歴駅を一覧表示する
2. 駅を直接確定する

### 検索起点
1. キーワード入力
2. `searchStationFuzzy()` の結果を表示
3. 駅を直接確定する

### 検索結果の整形
- 検索結果の各行には `displayName` を導出して保持する。
- `displayName` は `name` と `samename` の先頭有効値から生成する。
- `samename` は生データ起源の括弧付き値として扱い、括弧正規化は一覧表示と同じ共通ロジックで行う。
- かなは `searchStationFuzzy()` の返却値を優先し、空なら `getKanaByStation(displayName)` で補う。
- 都道府県は `getPrefectureByStation(displayName)` で取得し、失敗時のみベース名で再照会する。
- 検索結果カードでも都道府県は 1 行目右端に表示し、直前行と同一都道府県なら省略する。
- 並び順は `score` 昇順、同点時は `displayName` のロケール比較とする。
- 表示件数は最大 50 件とする。

## 発駅選択時の確定動作
- 必要に応じて既存経路上書き確認を行う。
- 新しい `Farert` を生成する。
- `addStartRoute(station)` を実行する。
- `mainRoute` を置き換える。
- `stationHistory` に追加する。
- メイン画面へ戻る。

### 発駅選択時の駅名
- `addStartRoute()` に渡す駅名は、元の駅名ではなく解決済み表示名とする。
  - 例: `追分` ではなく `追分(室)`
- 履歴にも同じ解決済み表示名を保存する。

## 着駅選択時の確定動作
- 選択直後に「新幹線を利用しますか？」確認ダイアログを出す。
- 確認後に `mainRoute.autoRoute(useBulletTrain, destination)` を実行する。
- 成功時:
  - `stationHistory` に追加する
  - メイン画面へ戻る
- 失敗時:
  - `mainScreenErrorMessage` にエラーを設定する
  - メイン画面へ戻る

### 着駅選択時の駅名
- `autoRoute()` に渡す駅名も解決済み表示名とする。
- したがって検索結果、駅一覧、履歴のどこから選んでも route へ渡る文字列は同一規則であること。

## 履歴管理
- `stationHistory` を最近順に表示する。
- 個別削除に対応する。
- 編集モードまたはスワイプ操作で削除する。
- 必要なら全消去導線を持てる。

## エラーと戻る挙動
- 検索中は戻るで検索解除を優先する。
- 広幅 2 ペインで駅一覧が表示中なら、戻るで右ペインの駅一覧だけを閉じる。
- 駅一覧表示中は戻るで路線一覧へ戻る。
- 路線一覧表示中は戻るでタブの root へ戻る。
- root 状態で戻るとメイン画面へ戻る。
- 一覧取得失敗、検索失敗時は画面内エラーバナーで通知する。

### 履歴表示の補足
- 履歴は保存済みの駅名文字列をそのまま表示する。
- 履歴項目は即時選択可能で、別途同名駅再解決は行わない。
- 履歴削除は個別削除を優先し、キーボードまたはスワイプ操作を実装してよい。
