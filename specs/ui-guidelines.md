# UI ガイドライン

## 基本
- 共通色は `src/app.css` の CSS 変数を使う。
- ボタン・カード・ダイアログは画面内で完結する素朴な実装を優先する。
- アイコンは `Material Symbols Rounded` を使う。

## エラー表示
- 画面上部または対象セクション直下にインライン表示する。
- 一時成功メッセージも同様にバナーで表示する。
- 画面遷移後のエラー返却には `mainScreenErrorMessage` を使う。

## ダイアログ
- 保存画面、発着駅選択画面、メイン画面で確認ダイアログを使う。
- ブラウザ標準ダイアログではなく、ページ内モーダルとして描画する。

## リスト
- 保存済み経路、履歴、駅一覧、路線一覧は単純な縦並び。
- きっぷホルダのみ並べ替え可能。

## きっぷホルダ
- PC は drag and drop、モバイルは touch 操作で並べ替える。
- `FarePicker` はカード内操作でもカード選択を誤発火させない。

## テーマ
- ライト/ダークを実装済み。
- トグル結果は `localStorage.theme` に保存する。

## ナビゲーション
- メイン画面と詳細画面の遷移は横方向スライド。
- その他の画面では複雑な遷移アニメーションを前提にしない。
