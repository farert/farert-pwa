# あいまい検索仕様

## 目的
- 発着駅選択で、駅名の表記ゆれ、かな入力、空白混入、一部誤字に対して候補駅を返す。
- 検索アルゴリズムの正本は WASM 側に置き、PWA は同じ検索 API を使う。

## 利用画面
- `/terminal-selection`

## 責務分担

### WASM 側の責務
- 正規化
- 候補探索
- スコアリング
- 並び順決定

### UI 側の責務
- 入力タイミングの制御
- 結果リスト表示
- 候補タップ時の確定
- かなや都道府県など補助情報の見せ方

## API 契約
- 呼び出し: `searchStationFuzzy(keyword, limit?)`
- PWA 側では返却結果を整形し、必要に応じて `getKanaByStation()` と `getPrefectureByStation()` で補足する。

### 返却レコード
候補 1 件は少なくとも次の情報を持つ。

- `name: string`
  - 駅のベース名。例: `柏原`
- `kana?: string`
  - 候補自身にかなが含まれる場合の優先かな
- `samename?: string[]`
  - 同名駅識別のための補助サフィックス群
  - 例: `["東"]`
  - データ起源により `"(東)"` や `"((東))"` のように括弧付きで来る場合がある
- `score?: number`
  - 小さいほど優先

UI は `name` をそのまま確定用キーとして扱ってはならず、必要に応じて `samename` を使って表示名を組み立てる。

## ユースケース
- `おち ゃの水` -> `御茶ノ水`
- `茅ケ崎` -> `茅ヶ崎`
- `竜が崎` -> `龍ヶ崎`
- `總持寺` -> `総持寺`
- `わしのす` -> `鳩ノ巣` のようなかな寄せ候補

## 検索キー
候補 1 件につき、少なくとも以下を検索キーとする。
- `name`
- `kana`
- `samename`

### 同名駅候補の重複排除
- WASM 側は `name` 単独ではなく、`name + samename + kana` 相当の組み合わせで完全重複を除去する。
- `name` が同じでも `samename` が異なる候補は残す。
  - 例: `柏原(東)` と `柏原(関)` は別候補
- `samename` が空でも `kana` が異なる候補は残す。
- UI 側で改めて `name` 単位の重複排除をしてはならない。

## 正規化の考え方
- 空白や改行を除去する
- 記号ゆれを吸収する
- かな揺れを吸収する
- 主要な異体字や代表文字へ寄せる

PWA 側ではこの正規化ロジックを持たず、仕様上も WASM 正本とする。

## マッチング手順
1. 正規化一致
2. `name / kana / samename` の横断候補化
3. 既存キーワード検索系候補の統合
4. かな末尾一致などの補助候補追加

## スコアリング
- 小さい値ほど優先度が高い。
- UI 側では score の意味を再解釈せず、返却順を尊重する。
- 同点時の安定順序も WASM 側が担う。

## UI 表示

### 候補 1 行の最小情報
- 駅名
- かな
- 都道府県

### 表示名の生成規則
- UI は候補表示用に `displayName` を導出する。
- `displayName` は `name` と `samename` の先頭有効値から組み立てる。
- `samename` は表示前に単一の丸括弧へ正規化する。
  - `東` -> `(東)`
  - `(東)` -> `(東)`
  - `((東))` -> `(東)`
- 最終表示名は `normalize(name) + normalizedSuffix` とする。
  - 例: `柏原` + `((東))` -> `柏原(東)`
- `samename` が空なら `displayName = name`

### かな・都道府県の補完順
1. `searchStationFuzzy()` の返却 `kana` を優先使用する。
2. `displayName` を使って `getKanaByStation(displayName)` / `getPrefectureByStation(displayName)` を照会する。
3. 失敗した場合のみ、サフィックス除去後のベース名で再照会する。
4. 再照会にも失敗したら空文字として扱う。

### 表示方針
- 候補タップで即確定する。
- 自動確定はしない。
- 誤字救済候補も通常候補と同じく明示的に選ばせる。
- 結果ゼロ件時は空状態メッセージを表示する。
- アクセシブル名と可視ラベルは同じ `displayName` を使う。

## 実装上の注意
- 検索入力中に都度 API を呼ぶ。
- UI 側は古い検索要求の結果で新しい結果を上書きしない。
  - 入力ごとにトークンを振り、最後に開始した検索だけを反映する。
- UI 側で `score` を再解釈しない。
- 同点時のみ `displayName` 昇順で安定化してよい。
- API の limit は UI 側の描画負荷を抑えるための制御であり、検索品質の制御ではない。
- 検索仕様変更時は `../farert-wasm` と合わせて更新する。
