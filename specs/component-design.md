# コンポーネント設計

## 方針
- 画面固有の状態遷移は `src/routes` に残し、再利用価値が明確な UI だけを `src/lib/components` に切り出す。
- コンポーネントは表示責務を中心に持ち、ルーティングや永続化の判断は持ち込まない。
- WASM 依存のオブジェクトは画面側で組み立て、コンポーネントには派生済みデータやコールバックを渡す。

## 主要コンポーネント

### `DrawerNavigation.svelte`
- 役割: きっぷホルダドロワー全体の表示と操作を担う。
- 責務:
  - 開閉状態の UI 表示
  - 合計運賃・営業キロの集計表示
  - 共有、編集切替、現在経路の追加
  - 並べ替え UI とカード列の管理
- 依存:
  - `TicketHolderCard.svelte`
  - ドラッグ・タッチ操作
- 画面側に残す責務:
  - 経路を開く前の確認ダイアログ
  - `mainRoute` への反映
  - 共有リンク生成
  - `Farert` を使った表示値再計算

### `TicketHolderCard.svelte`
- 役割: きっぷホルダ 1 件分の表示単位。
- 責務:
  - 経路名と `routeScript` 相当の情報表示
  - 運賃種別の切替
  - 編集モード時の削除導線
  - 並べ替えハンドル表示
- 注意点:
  - カード全体タップと `FarePicker` の操作が衝突しないようにする。
  - モバイルでは誤タップを避けるため、編集モードを明示的に切り替えてから削除する。
  - ドラッグハンドル以外の領域で並べ替え開始しない。

#### 入出力の考え方
- 入力:
  - タイトル
  - 運賃表示文字列
  - キロ表示文字列
  - 運賃種別
  - 削除可否
  - ドロップ位置
- 出力:
  - 選択
  - 削除
  - 運賃種別変更
  - 並べ替え開始 / 継続 / 確定

### `FarePicker.svelte`
- 役割: `FareType` の選択 UI に責務を限定する。
- 責務:
  - 表示中運賃種別の視覚化
  - 変更イベントの通知
- 非責務:
  - 運賃再計算
  - route 保存
  - 親カード選択

#### 注意点
- `click`, `pointerdown`, `focusin` は親カードへ伝播させない。
- 運賃種別の選択肢自体は UI 側 enum であり、WASM のオプション設定とは別物である。

### `FareSummaryCard.svelte`
- 役割: メイン画面の簡易サマリー表示。
- 責務:
  - 普通運賃
  - 営業キロ
  - 有効日数
  - 詳細画面への導線
- 前提:
  - WASM から取得した `fareInfo` は親で整形済みであること

### `SavedRouteCard.svelte`
- 役割: 保存画面で現在経路または保存済み経路を 1 件表示する。
- 責務:
  - route 表示
  - 読込、共有、削除導線
  - 編集モード時の削除 UI
- 親画面との差分:
  - インポート / エクスポート / クリップボード制御は保存画面側で持つ

## 親子責務の原則
- 親が持つもの:
  - WASM 呼び出し
  - 永続化
  - 画面遷移
  - 確認ダイアログ
  - 表示用 ViewData 生成
- 子が持つもの:
  - 見た目
  - ユーザーイベントの発火
  - 局所的な UI 状態

## 画面内で完結している主要 UI
- メイン画面:
  - 上部バー
  - 発駅カード
  - 区間カード
  - 下部ナビゲーション
- 詳細画面:
  - メトリクスカード
  - 区間別情報
  - 共有ボタン群
- 保存画面:
  - 確認モーダル
  - 手動入力インポート UI
- 発着駅選択画面:
  - タブ
  - 検索バー
  - 履歴スワイプ UI

## 分割基準
- 複数画面から使う、または単独でテスト価値があるものは `src/lib/components` に置く。
- その画面の URL 文脈や store 更新を前提にするものは route 直下に置く。
- 共通部品へ切り出すことで逆に props が肥大化する場合は、画面内ローカル実装のままにする。
